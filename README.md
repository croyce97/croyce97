## Engineering principles

- Prefer simple systems that are easy to operate under pressure.
- Automate repeatable work, but keep the failure mode understandable.
- Treat observability, rollback, and security as part of delivery, not afterthoughts.
- Write documentation that helps another engineer recover the system at 2 AM.
